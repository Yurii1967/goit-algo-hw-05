## Завдання 3. Висновки

У межах третього завдання було реалізовано та експериментально порівняно три алгоритми пошуку підрядка в тексті:
**Боєра–Мура**, **Кнута–Морріса–Пратта** та **Рабіна–Карпа**.

Для аналізу використовувалися два різні за обсягом та структурою тексти. Для кожного тексту вимірювався час пошуку:
- підрядка, який існує в тексті;
- вигаданого підрядка, який у тексті відсутній.

Час виконання алгоритмів вимірювався за допомогою модуля `timeit`.

### Результати для тексту 1
Для першого тексту (більшого за обсягом) найшвидшим виявився **алгоритм Боєра–Мура**.  
Це пояснюється використанням евристик, які дозволяють виконувати значні стрибки по тексту та зменшувати кількість порівнянь символів.

Алгоритм Кнута–Морріса–Пратта продемонстрував стабільну, але дещо повільнішу роботу, а алгоритм Рабіна–Карпа показав найгірший результат через додаткові витрати на обчислення хешів.

### Результати для тексту 2
Для другого, меншого за обсягом тексту, найкращу продуктивність продемонстрував **алгоритм Кнута–Морріса–Пратта**.  
У цьому випадку переваги евристик Боєра–Мура були менш помітними, тоді як KMP забезпечив рівномірну та передбачувану швидкість роботи.

Алгоритм Рабіна–Карпа знову поступився іншим за швидкістю.

### Загальний висновок
За результатами експериментів можна зробити такі узагальнення:
- **алгоритм Боєра–Мура** є найефективнішим для пошуку підрядків у великих текстах;
- **алгоритм Кнута–Морріса–Пратта** демонструє стабільну продуктивність незалежно від структури тексту;
- **алгоритм Рабіна–Карпа** в практичних тестах показав найнижчу ефективність.

Отже, найшвидшим алгоритмом **у цілому** для проведених експериментів є **алгоритм Боєра–Мура**.
